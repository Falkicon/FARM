Typed environment configuration Provides validated and transformed environment variables

Type declaration

  • ReadonlyisDevelopment: boolean

    Whether running in development mode

  • ReadonlyisTest: boolean

    Whether running in test mode

  • ReadonlyisProduction: boolean

    Whether running in production mode

  • Readonlyapp: { port: number; frontendUrl: string }

    Application configuration

    • Readonlyport: number

      Server port number

    • ReadonlyfrontendUrl: string

      Frontend application URL

  • Readonlydatabase: { url: string }

    Database configuration

    • Readonlyurl: string

      Active database URL (uses TEST_DATABASE_URL in test environment if available)

  • Readonlysecurity: { corsOrigins: string[] }

    Security configuration

    • ReadonlycorsOrigins: string[]

      Allowed CORS origins

  • Readonlylogging: { level: "info" | "warn" | "error" | "debug"; format: "pretty" | "json" }

    Logging configuration

    • Readonlylevel: "info" | "warn" | "error" | "debug"

      Log level (debug, info, warn, error)

    • Readonlyformat: "pretty" | "json"

      Log format (json, pretty)

Example

// Check environment
if (config.isDevelopment) {
  console.log('Running in development mode');
}

// Access configuration
const serverPort = config.app.port;
const dbUrl = config.database.url;
const corsOrigins = config.security.corsOrigins;

Remarks

Features:

  • Type-safe configuration access
  • Environment-specific values
  • Validated using Zod schema
  • Immutable configuration object
